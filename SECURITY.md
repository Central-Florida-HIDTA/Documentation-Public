# Security Policy

## Reporting a Vulnerability

Please report security issues privately using GitHub Security Advisories or by contacting the repository owner directly.

## Sensitive Data Policy

This repository must not contain:

- Credentials, API keys, tokens, or client secrets
- Real tenant identifiers that are not intended for public disclosure
- Personal identifying information
- Internal incident evidence or audit exports

If sensitive data is committed accidentally:

1. Rotate affected credentials immediately.
2. Remove sensitive content from current files.
3. Rewrite repository history if needed.
4. Document remediation in an internal incident record.
